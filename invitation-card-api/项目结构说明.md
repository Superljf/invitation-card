# invitation-card-api 项目结构说明

## 目录概览

```
invitation-card-api/
├── pom.xml                    # Maven 配置
├── README.md                   # 快速上手
├── 项目结构说明.md             # 本说明文档
└── src/main/
    ├── java/com/invitation/
    │   ├── InvitationCardApplication.java   # 启动类
    │   ├── config/              # 配置
    │   ├── controller/          # 控制器
    │   ├── dto/                 # 数据传输对象
    │   ├── model/               # 模型/实体
    │   └── service/             # 业务逻辑
    └── resources/
        └── application.yml      # 应用配置
```

---

## 各文件夹说明

### config（配置层）

**作用**：存放各种配置类，对应用全局行为进行配置。

- **CorsConfig.java**：跨域配置，允许前端（如 localhost:5173）调用本后端 API，解决浏览器的跨域限制。

---

### controller（控制器层）

**作用**：接收 HTTP 请求，调用 service，并返回 HTTP 响应。对应 REST API 的入口。

- **InvitationController.java**：请柬相关接口
  - `POST /api/invitations`：保存请柬
  - `GET /api/invitations/{id}`：按 id 获取请柬

---

### dto（数据传输对象 / Data Transfer Object）

**作用**：定义接口的请求、响应结构，用于前后端或服务间传递数据。字段和前端约定的数据结构一致。

| 文件 | 说明 |
|------|------|
| **FormDataDto.java** | 请柬编辑数据，对应前端 `FormData`（收件人、新郎、新娘、日期、地点等） |
| **SaveRequest.java** | 保存接口的请求体：`{ formData, templateId }` |
| **SaveResponse.java** | 保存接口的响应：`{ id }` |
| **InvitationResponse.java** | 获取接口的响应：`{ id, formData, templateId }` |

---

### model（模型/实体层）

**作用**：定义内部使用的数据结构，多用于内存/数据库存储。

- **InvitationRecord.java**：请柬记录，包含 `id`、`formData`、`templateId`，用于在内存 Map 中保存和读取。

---

### service（业务逻辑层）

**作用**：实现具体业务逻辑，被 controller 调用。不直接处理 HTTP，便于复用和测试。

- **InvitationService.java**：请柬相关业务
  - `save()`：生成 id，将请柬存入内存 Map
  - `get()`：按 id 从内存 Map 中取出请柬

---

## 请求流转示意

```
前端 fetch → Controller（接收、校验）→ Service（业务处理）→ 返回 JSON
```

---

## 数据流说明

1. **保存请柬**：前端 POST 提交 `SaveRequest` → Controller 解析 → Service 存入 Map → 返回 `SaveResponse`（包含 id）
2. **获取请柬**：前端 GET 带 id → Controller 转给 Service 查询 → 返回 `InvitationResponse`（formData + templateId）
